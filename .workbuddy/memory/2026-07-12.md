## 诊断：会话「查找word skill」为何不能执行命令
- 两条相关会话：`~/.maestro/sessions/2c022fb404054999abaca4fdf2542f39.json`（「查找word skill」）、`8336ea55…json`（「帮我查找word的skill」）。Agent 把 `npx --yes skills find word` 当文本输出并声明"无法执行命令行搜索"。
- 三层根因：
  1. **命令是幻觉**：环境里没有 `skills` 这个 npm/cli，`npx --yes skills find` 是 LLM 凭空生成的，无真实程序可跑。
  2. **Agent 没有"查技能市场"的工具**：目录只能经 HTTP API `GET /extension-catalog/skills` 查询，但未接成 agent 工具；`tool_search` 只搜 Agent 自身已注册工具、`bash` 也够不到市场，所以 LLM 无对应能力→退化成给假命令。
  3. **意图越界**："查 word 技能"非排产任务，编排器判不确定/非调度，系统提示限定排产领域→不愿乱调 bash。
- **更要命：目录里根本没有 Word 技能**。`sources.py:6` 的 anthropics `paths=["skills/pdf","skills/slides","skills/spreadsheets","skills/docs"]` 已与上游脱节——上游 `anthropics/skills` 现结构是 `skills/docx`(Word)、`skills/pdf`、`skills/pptx`、`skills/xlsx`，**没有 docs/slides/spreadsheets**。因此只有 `pdf` 进了目录（且许可证 Proprietary 不可安装）。真正的 Word 技能 `skills/docx` 不在 allowlist，从未同步。搜 `word` 返回 0。
- 修复方向（待用户确认再做）：①`sources.py` anthropics paths 改为 `["skills/pdf","skills/docx","skills/pptx","skills/xlsx"]` 后重新同步；②给 Agent 接一个"查扩展目录+安装/添加"的工具并放开权限。
