Maestro Design System · v2 · Draft

深空指挥舱

制造执行 Agent 的双主题视觉系统。深空（Dark）为默认主场，极昼（Light）是同级公民——同一套 HUD 秩序，两种光照。本页即设计稿：所有色值、材质、组件、界面均由 CSS 变量实时渲染，右上角可切换主题。

01 / Concept

设计概念与原则

工厂不是办公室，是值班现场。Maestro 的界面是一座「指挥舱」：操作员面对的是一个会思考、会请求授权的执行体——界面必须让人一眼看清 它在做什么、做到哪一步、哪一步需要我拍板。

原则含义界面体现
深色为主场值班场景长时间注视，深色降低眩光；霓虹语义色在暗底上获得最高信噪比。深空默认；浅色（极昼）不是反色，是重设计的同级主题。
一色一职颜色只承担语义，不做装饰。全App同一颜色永远代表同一件事。青=沟通与导航 · 绿=执行与成功 · 琥珀=等待你确认 · 红=失败与危险 · 紫=受控执行路径。
模式四色「谁接单」独立于「能不能做」：模式轴（自动/排产/调度/查询）与授权轴（绿/琥珀/红）正交，互不借色。自动=青 · 排产=蓝 · 调度=橙 · 查询=青绿；只出现在模式分段、徽标与徽点。
玻璃分层信息分三层：虚空（底）→ 舱体（实心）→ 玻璃（悬浮）。层级靠材质而非投影堆叠。对话流在虚空上，任务卡/输入条是玻璃，轨迹坞是舱体。
HUD 秩序等宽字体做标签与数据、切角与括号做聚焦、网格做底色——秩序感来自工程制图，不是科幻贴纸。JetBrains Mono 微标签 · HUD 角括号 · 蓝图网格 · 切角按钮。
动效有任务动画只表达状态转移（流式、等待、升级、完成），不做装饰性表演；可被系统减动效关闭。扫描线=正在生成 · 脉冲点=等待审批 · 全部遵守 prefers-reduced-motion。

03 / Typography

字体与排版

三轨字体：Space Grotesk 承担拉丁标题与数字（几何、工程感），JetBrains Mono 承担 HUD 微标签与一切数据/编号（等宽即秩序），中文交给系统黑体。中文字重封顶 500、字距归零——PingFang SC 无真 SemiBold，600 会触发伪粗体。本页内嵌两款拉丁字体的 latin 子集，离线可用。
