# 排产与调度基础概念

## 什么是排产
排产 (Planning) 指在交期、产能、物料、工艺约束下，为一批生产订单求解出「在哪条产线、按什么顺序、何时开工」的生产计划。排产是一次性的优化求解，输入是订单集合与约束，输出是任务令的时间表与产线分配。

## 排产与调度的区别
排产解决「计划怎么排」，是离线优化；调度 (Scheduling) 解决「计划怎么落地执行」，是在线响应。调度关注齐套检查、催料、任务令下发、异常处置等执行环节。计划排好后，调度负责让它真正按计划跑起来，并在缺料、设备故障等扰动发生时实时应对。

## 任务令状态流转
任务令 (Work Order) 的状态: draft (待下发) → dispatched (已下发) → in_progress (生产中) → done (完工)。物料不齐或前道未完成时可能处于 blocked (阻塞)。只有 draft 且已齐套、产线可用的任务令才允许下发。

## 常见排产目标
- 最小化拖期 (tardiness): 尽量让订单不晚于交期完成，适合交期敏感场景。
- 最小化完工跨度 (makespan): 尽快做完整批订单，适合产能优先场景。
- 简单派工 (dispatch rule): 按规则 (如最早交期优先 EDD) 快速排，适合扰动多、计划频繁变更的场景。
